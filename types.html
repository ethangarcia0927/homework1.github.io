High-Level vs. Low-Level Languages

Programming languages are often classified by how closely they interact with hardware. Low-level languages, like Assembly, Communicate directly with the computer's processor. High-level languages, like Python and Java, are easier for humans to read and write because they use more natural language elements

Compiled vs. Interpreted Languages

A compiled language is converted into machine code before it runs. Examples include C and C++. These are fast and efficient but require a compilation step.

In contrast, an interpreted language executes instructions line by line at runtime. Python and JavaScript are interpreted, which makes development easier.

Programming Paradigms

Programming paradigms describe how code is structured and executed. The main paradigms are:

Procedural: Focuses on how a program operates. It uses explicit, step-by-step commands that directly change the state of the program. It tells the computer exactly what to do and how to do it.

Object-Oriented: Organizes code around data, or objects, rather than fucntions. Objects represent entities and combine both data (attributes) and behavior (methods) into a single unit. It models the problem domain in terms of interacting objects.

Functional: Treats computation as the evaulation of mathematical fuynctions and avoids changing state or mutable data. It emphasizes immutaability, pure functions, and expressions.

Logic: Based on formal logic and a set of facts and rules. THe programmer specifies the problem in terms of known facts and relationships, and the language's system uses an inference engine to find solutions or answer queries.
