High-Level vs. Low-Level Languages

Programming languages are often classified by how closely they interact with hardware. Low-level languages, like Assembly, communicate directly with the computer’s processor. High-level languages, like Python and Java, are easier for humans to read and write because they use more natural language elements.

Compiled vs. Interpreted Languages

A compiled language is converted into machine code before it runs. Examples include C and C++. These are fast and efficient but require a compilation step.

In contrast, an interpreted language executes instructions line by line at runtime. Python and JavaScript are interpreted, which makes development easier.

Programming Paradigms

The main paradigms describe how code is structured and executed:

Procedural

Focuses on how a program operates: step-by-step commands that change program state explicitly.

Object-Oriented

Organizes code around data, or objects, rather than functions. Objects bundle data (attributes) with behavior (methods) to model real entities.

Functional

Treats computation as the evaluation of mathematical functions and avoids changing state or mutable data. Emphasizes immutability and pure functions.

Logic

Based on formal logic with facts and rules. The programmer states relationships; an inference engine derives answers or solutions.

Did You Know?

Many modern languages, like Python and Swift, combine multiple paradigms—they can be both object-oriented and functional.
